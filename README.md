# Module-Eight-Journal
Completed software design document project

Client Summary
The client, The Gaming Room, wanted a web-based version of their game Draw It or Lose It. The goal was to create a system that supports multiple teams and players while making sure that all names are unique. The application also needed to run across different platforms and allow multiple users to play at the same time. To meet these requirements, I designed a scalable system using object-oriented principles and design patterns like Singleton and Iterator.

What I Did Well
I think I did a good job organizing the design document and clearly explaining how each part of the system works. My domain model was strong because it showed the relationships between classes like Game, Team, and Player in a simple way. I also did well explaining design patterns and why they were used, especially how the Singleton pattern helps manage memory and keep only one instance of the game service.

What Helped During Development
Working through the design document helped me understand the structure of the program before writing code. It made it easier to break the system into smaller parts and focus on one piece at a time. Having a clear plan also helped avoid confusion later and made the implementation process more efficient.

What I Would Improve
If I could revise one part, I would improve the system architecture section by adding more detailed diagrams or visuals. While the explanations were clear, diagrams would make it easier to understand how everything connects, especially for someone new to the project.

Understanding User Needs
I interpreted the user’s needs by focusing on scalability, ease of use, and performance. For example, ensuring unique names and supporting multiple players directly reflects what the client wanted. Considering user needs is important because it ensures the software is actually useful and solves the problem it was designed for. Without that, the system might work technically but fail in real-world use.

My Design Approach
When designing the software, I focused on object-oriented design, breaking the system into classes with clear responsibilities. I also used design patterns to improve structure and maintainability. In the future, I would continue using these strategies, along with techniques like UML diagrams, prototyping, and user feedback, to design similar applications more effectively.
